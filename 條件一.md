# 校園租借系統 - User Story 功能映射

根據 PDF 中的 User Story，以下是系統實現的功能映射：

## Epic 1: 借用流程與通知

### 1.1 線上借用申請 ✅
- **接受條件 Scenario 1**: 成功提交借用申請
  - ✓ 用戶填寫借用表單（申請人、時間、用途等）
  - ✓ 系統驗證所有必填欄位
  - ✓ 系統顯示「申請成功」訊息
  - ✓ 申請資料記錄於系統中，等待管理員審核

- **接受條件 Scenario 3** (異常處理): 未填寫必要欄位
  - ✓ 系統驗證所有必填欄位
  - ✓ 顯示「請填寫所有必填欄位」的錯誤訊息
  - ✓ 要求補充資料後再提交

### 1.2 借用狀態即時查詢 ✅
- ✓ 用戶可在「我的申請」中查看所有申請
- ✓ 實時顯示申請狀態（待審核、已通過、已拒絕）
- ✓ 支持按狀態篩選

### 1.3 活動企劃書數位遞交 ✅
- ✓ 申請表單支持文件上傳
- ✓ 支持 PDF、DOC、DOCX 格式

### 1.4 申請案暫存功能 ✅
- **接受條件 Scenario 1**: 成功儲存草稿
  - ✓ 用戶可點擊「暫存草稿」按鈕
  - ✓ 系統顯示「草稿已儲存」訊息
  - ✓ 資料暫存在個人專區

- **接受條件 Scenario 2**: 重新載入草稿
  - ✓ 用戶可從「我的申請」重新打開草稿
  - ✓ 系統自動帶入上次填寫的資料

### 1.8 補件/修改機制 ✅
- ✓ 拒絕的申請可直接在原申請單上修改
- ✓ 修改後重新提交

### 1.9 申請狀態變更即時通知 ✅
- **接受條件 Scenario 1**: 申請通過通知
  - ✓ 系統發送「您的申請已通過審核」通知
  - ✓ 包含所有相關詳情

- **接受條件 Scenario 2**: 申請被退回通知
  - ✓ 系統發送「您的申請已被退回」通知
  - ✓ 附上退回原因和修改提示

- **接受條件 Scenario 3**: 補件提醒通知
  - ✓ 系統發送補件提醒訊息
  - ✓ 提供需補充資料的說明

### 1.13 器材報修回報 ✅
- **接受條件 Scenario 1**: 成功提交損壞回報
  - ✓ 用戶填寫故障報告表單
  - ✓ 系統顯示「回報成功」訊息
  - ✓ 回報內容記錄於系統

- **接受條件 Scenario 2**: 報修時附上照片
  - ✓ 支持上傳照片
  - ✓ 系統儲存照片與回報內容

## Epic 2: 空間與器材整合管理

### 2.1 資源資訊與可用性管理 ✅
- ✓ 課指組可新增/編輯資源資訊
- ✓ 設定資源可借用狀態
- ✓ 禁用/啟用資源

### 2.2 一鍵核准/退回 ✅
- **接受條件**: 快速審核機制
  - ✓ 課指組人員一鍵快速核准申請
  - ✓ 一鍵拒絕申請
  - ✓ 支持多筆勾選操作（可擴展）
  - ✓ 提升行政流程效率

### 2.3 自動檢查與鄰近時段鎖定 ✅
- ✓ 系統自動檢查申請時間
- ✓ 檢測時間衝突
- ✓ 衝突時自動鎖定相鄰時段（提前/延後30分鐘）

### 2.4 審核範本（罐頭訊息） ✅
- ✓ 提供常見拒絕原因選單
- ✓ 快速選擇原因，無需重複打字

### 2.5 逾期未還與違規自動催繳 ✅
- **接受條件 Scenario 1**: 器材逾期未歸還自動催繳
  - ✓ 系統定時檢查借用紀錄
  - ✓ 超過歸還時間發送催繳通知

- **接受條件 Scenario 2**: 同步通知社團負責人
  - ✓ 系統同時將通知副本發送給社團指導老師

- **接受條件 Scenario 3**: 避免重複通知
  - ✓ 系統依設定時間間隔發送
  - ✓ 避免短時間內重複發送

### 2.6 借用紀錄查詢 ✅
- ✓ 課指組可查看全校歷史借用紀錄
- ✓ 支持按申請人、時間、資源篩選

### 2.7 資源統計分析 ✅
- ✓ 顯示統計信息
- ✓ 可匯出 Excel/PDF 報表（可擴展）

## 使用說明與測試場景

### 測試場景 1: 學生提交申請
1. 登入系統，點擊「我要租借」
2. 選擇資源，填寫申請表
3. 點擊「暫存草稿」測試草稿功能
4. 重新開啟並完成填寫
5. 提交申請，查看成功訊息

### 測試場景 2: 課指組審核
1. 以課指組角色登入
2. 進入「審核面板」
3. 查看待審核申請
4. 快速核准或拒絕（含自動通知）
5. 查看「審核紀錄」

### 測試場景 3: 時段衝突檢測
1. 嘗試提交與已核准申請時間重疊的申請
2. 系統應顯示衝突提示
3. 觀察相鄰時段是否被自動鎖定

### 測試場景 4: 故障報修
1. 在「我的申請」中的已核准申請旁點擊「報修」按鈕
2. 填寫故障描述（可附圖片）
3. 提交報修，驗證通知發送
